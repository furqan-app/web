# Retrospective — 2026-07-01

## Session Summary

Fixed verse text rendering outside the Quran page route across four surfaces: search results, MarkModal word title, MarkModal verse title, and the rub list start verse. Root cause was `verse.text_uthmani` being rendered directly, which contains rub el hizb markers (۞ U+06DE) that the previous font (`uthmanic.ttf`) could not render. All surfaces now use `word.qpc_uthmani_hafs` reconstructed via word-join, with `UthmanicHafs1Ver18` as the global non-page font.

## Changes Made

- [x] Updated DECISIONS.md: clarified "Verse has no qpc_uthmani_hafs" constraint — prefer word reconstruction over `text_uthmani` when words are in scope
- [x] Saved memory: consolidate related bugs into existing plans rather than creating new plan files

## Flagged for Later

- The `endVerse` select in `RubWithVerses` / `get-rubs.ts` still fetches `text_uthmani` but it is never rendered. Could be removed if no future use is planned.

## Open Considerations

- ADR 0002's "Affected Surfaces" table should be kept up to date as new non-page Quran text rendering contexts are added. Any new component that displays verse or word text outside the page route must follow the word-join pattern.
